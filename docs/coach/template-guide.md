# Program Template Guide

How Sthir templates work and how loads are calculated.

## Template selection

Engine: `src/lib/program-engine.ts` → `selectTemplate(answers)`

Inputs:
- **goal** — primary selector (8 goals)
- **experience** — beginner through advanced
- **federation** — tags filter WRPF vs IPF variants
- **meetDate** — meet prep templates if date within window

## Available goals → templates

| Goal ID | Typical duration | Focus |
|---------|------------------|-------|
| `first_meet` | 12 weeks | Meet prep, peaking, attempt strategy |
| `increase_total` | 12–16 weeks | Balanced SBD progression |
| `improve_squat` | 8–12 weeks | Squat frequency + accessories |
| `improve_bench` | 8–12 weeks | Bench volume + weak points |
| `improve_deadlift` | 8–12 weeks | Deadlift technique + posterior chain |
| `powerbuilding` | 12 weeks | Strength + hypertrophy hybrid |
| `general_strength` | 12 weeks | Balanced block |
| `fat_loss_strength` | 8 weeks | Moderate volume, maintain intensity |

## Block structure

Each program = array of `ProgramBlock`:

```
week, day, exercise, sets, reps, load, rpe, notes
```

Generated by `generateProgramBlocks(template, answers)`.

## Load calculation

1. **Base:** athlete's 1RM for exercise (squat/bench/deadlift or derived)
2. **Template intensity:** `intensityPct` per exercise (e.g., 72%)
3. **Phase multiplier:** accumulation (1.0) → intensification (0.95) → peaking (0.85–0.90)
4. **Formula:** `loadKg = roundToPlate(max * intensityPct * phaseMult / 100)`
5. **Rounding:** nearest 2.5 kg

### Training style formatting

| Style | Load string example |
|-------|---------------------|
| `percentage` | `120 kg (75%)` |
| `rpe` | `@ RPE 8 (~120 kg)` |
| `mixed` | `120 kg (75%) / RPE 8` |

Bodyweight/accessory: `"Bodyweight / RPE only"`

## Phase multipliers

Template defines week ranges:
- Weeks 1–4: accumulation (multiplier 1.0)
- Weeks 5–8: intensification (0.95)
- Weeks 9–12: peaking (0.85–0.90)
- Meet week: taper per template

Coach should verify peaking weeks against athlete's meet date.

## Coach notes generation

`generateCoachNotes(template, answers)` produces draft:

- Program name, athlete summary, SBD maxes
- Federation, training days, style
- Injury considerations (if any)
- Meet date taper reminder
- Template-specific notes (nutrition, meet day)
- Standard readiness rule

**Always personalize** before approve — see [review-sop.md](review-sop.md).

## Validation warnings

`validateLiftRatios(answers)` flags:
- Bench unusually low vs squat
- Deadlift unusually low vs squat
- Extreme totals for experience level

Warnings returned in intake API — not blockers, coach decides.

## Adding / editing templates

1. Open GitLab issue (`type::requirement`) — requires PM + CoachOps + Legal approval
2. Edit template definitions in `program-engine.ts`
3. Add/update unit tests in `program-engine.test.ts`
4. Update this guide + [federation-rules.md](federation-rules.md)
5. CHANGELOG entry

## Manual override (concierge)

If engine output wrong:
1. Export CSV from admin deliver
2. Edit in Google Sheets
3. Send edited sheet to athlete
4. Log issue for template fix
